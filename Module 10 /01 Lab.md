## Теоретическая справка

### 1. Мандатное управление доступом (МРД)

Мандатное управление доступом в Astra Linux обеспечивает защиту информации различных уровней конфиденциальности путем управления доступом к конфиденциальным данным .

**Основные понятия:**

| Термин | Описание |
|--------|----------|
| **Метка безопасности** | Набор атрибутов (уровень конфиденциальности, категории, уровень целостности), присваиваемый субъектам и объектам |
| **Уровень конфиденциальности** | Иерархический уровень (число от 0 до 255), определяющий степень секретности |
| **Категория конфиденциальности** | Неиерархическая маска (64-битная), определяющая тематическую принадлежность |
| **Уровень целостности (МКЦ)** | Мандатный контроль целостности — степень доверия к субъекту или объекту  |

При установке ОС по умолчанию предлагается диапазон уровней конфиденциальности от 0 до 3 и набор категорий 0, 1 и 2 .

### 2. Утилиты управления МРД

| Команда | Описание | Пример |
|---------|----------|--------|
| `pdpl-file` | Установка параметров мандатного доступа на файлы  | `sudo pdpl-file 3:0:0:0 file.txt` |
| `pdp-ls` | Просмотр информации о файлах с метками безопасности  | `pdp-ls -M /path` |
| `pdpl-user` | Управление допустимыми метками пользователей  | `sudo pdpl-user -i 63 username` |
| `userlev` | Просмотр и редактирование уровней доступа  | `userlev --help` |
| `pdpl-ps` | Чтение мандатного контекста процесса  | `pdpl-ps $$` |


### 3. Синтаксис pdpl-file

```bash
pdpl-file [Уровень_МРД]:[Уровень_МКЦ:[Категории_МРД:[Флаги]]] Имя_объекта
```

Где :
- **Уровень_МРД** — уровень конфиденциальности (имя или число, например, `3` или `Уровень_3`)
- **Уровень_МКЦ** — уровень целостности (имя или число)
- **Категории_МРД** — список категорий через запятую (например, `Категория_1,Категория_2`)
- **Флаги** — мандатные атрибуты (например, `ccnr`)

**Примеры :**
```bash
# Назначение уровня 3, целостности 0, без категорий
sudo pdpl-file 3:0:0:0 /path/to/file

# Назначение с именованными уровнями
sudo pdpl-file Уровень_3:Высокий:Категория_1,Категория_2:ccnr /path/to/file
```

### 4. Синтаксис pdpl-user

```bash
pdpl-user [-m минимальный:максимальный уровень] [-i максимальный уровень целостности] 
          [-c минимальная категория:максимальная категория] <пользователь>
```

**Примеры :**
```bash
# Установка уровня целостности 63 (высокий)
sudo pdpl-user -i 63 username

# Установка диапазона уровней и категорий
sudo pdpl-user -m Уровень_0:Уровень_3 -i 0 -c 0:Категория_2 user1
```

### 5. Проверка состояния подсистемы


# Проверка статуса МКЦ
sudo astra-mic-control status

# Проверка меток пользователя
pdpl-user username
```

## Решение (эталон выполнения)

```bash
# ============ ПРОВЕРКА ПРАВ ============
# Все команды управления метками требуют прав root
sudo -i

# ============ ЗАДАНИЕ 1 ============
# Проверка состояния PDP
pdpctl get-mode
# Ожидаемый вывод: enforcing

# Проверка статуса МКЦ
astra-mic-control status

# Проверка модуля ядра
lsmod | grep parsec

# Проверка поддержки xattr
mount | grep xattr


# ============ ЗААНИЕ 2 ============
cd ~
mkdir -p lab_mandatory
cd lab_mandatory

echo "Public information" > file1.txt
echo "Confidential data" > file2.txt
echo "Secret document" > file3.txt
echo '#!/bin/bash' > script.sh
echo 'echo "Script running"' >> script.sh
chmod +x script.sh

# Проверка текущих меток
pdp-ls -M

# ============ ЗАДАНИЕ 3 ============
# Назначение меток файлам
pdpl-file 0:0:0:0 file1.txt
pdpl-file 1:0:0:0 file2.txt
pdpl-file 2:0:0:0 file3.txt
pdpl-file 1:2:0:0 script.sh

# Назначение категории
pdpl-file 2:0:Категория_1:0 file3.txt

# Просмотр назначенных меток
pdp-ls -M

# ============ ЗАДАНИЕ 4 ============
# Просмотр с метками
pdp-ls -M

# Подробная информация о файле
pdp-ls -M file2.txt

# Сравнение вывода
ls -la
pdp-ls -M

# Проверка доступа к файлу (от имени sa)
cat file3.txt  # Если недоступен - объяснить причину

# ============ ЗАДАНИЕ 5 ============
# Просмотр уровней пользователя
userlev
usercat

# ============ ЗАДАНИЕ 6 ============
# Просмотр текущих меток пользователя
pdpl-user sa

# Установка высокого уровня целостности
pdpl-user -i 63 sa

# Проверка изменения
pdpl-user sa

# Установка диапазона уровней
pdpl-user -m 0:3 sa

# Установка категорий
pdpl-user -c 0:2 sa

# Итоговая проверка
pdpl-user sa

# ============ ЗАДАНИЕ 7 ============
echo "Top secret" > secret.txt
pdpl-file 3:0:Категория_0,Категория_1:0 secret.txt
pdp-ls -M secret.txt

mkdir classified
pdpl-file 2:0:0:0 classified/
pdpl-file -R 2:0:0:0 classified/
pdp-ls -M -R

# ============ ЗАДАНИЕ 8 ============
# Назначение флага ccnr
pdpl-file 1:0:0:ccnr file2.txt
pdp-ls -M file2.txt

# Объединение меток (unite)
pdpl-file -u 0:0:Категория_2:0 file2.txt
pdp-ls -M file2.txt

# Вычитание категорий (subtract)
pdpl-file -s 0:0:Категория_1:0 file2.txt
pdp-ls -M file2.txt

# ============ ЗАДАНИЕ 9 ============
# Создание пользователя с низким уровнем
useradd -m -s /bin/bash testuser
echo "testuser:Test123!" | chpasswd
pdpl-user -m 0:0 -c 0:0 testuser

# Проверка доступа (от имени testuser)
sudo -u testuser cat /home/sa/lab_mandatory/file2.txt
# Должна быть ошибка доступа

# Вывод: мандатное управление ограничивает доступ к объектам с более высокими метками

# ============ ЗАКЛЮЧИТЕЛЬНЫЙ ОТЧЕТ ============
{
    echo "=== Mandatory Access Control Report ==="
    echo "Date: $(date)"
    echo
    echo
    echo "=== User Labels (sa) ==="
    pdpl-user sa
    echo
    echo "=== Files in lab_mandatory with labels ==="
    pdp-ls -M -R ~/lab_mandatory/
    echo
    echo "=== Attributes (lsattr) ==="
    lsattr -R ~/lab_mandatory/
} > mrd_report.txt

cat mrd_report.txt
```
