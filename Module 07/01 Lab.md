## Теоретическая справка

### 1. Основные понятия о процессах

| Термин | Описание |
|--------|----------|
| **Процесс** | Экземпляр выполняющейся программы со своим адресным пространством |
| **PID** | Уникальный идентификатор процесса (Process ID) |
| **PPID** | PID родительского процесса |
| **UID/GID** | Идентификаторы пользователя и группы, от имени которых выполняется процесс |
| **Приоритет (NI)** | Значение (от -20 до 19), определяющее время CPU для процесса |
| **Состояние** | R (Running), S (Sleeping), D (Uninterruptible), Z (Zombie), T (Stopped) |

### 2. Мониторинг процессов

| Команда | Описание | Пример |
|---------|----------|--------|
| `ps` | Вывод информации о процессах | `ps aux` |
| `top` | Интерактивный мониторинг в реальном времени | `top` |
| `htop` | Улучшенный интерактивный мониторинг | `htop` |
| `pstree` | Дерево процессов | `pstree -p` |
| `pgrep` | Поиск PID по имени | `pgrep bash` |
| `pidof` | Определить PID по имени | `pidof systemd` |

**Опции ps:**
```bash
ps aux          # Все процессы в формате BSD
ps -ef          # Все процессы в формате System V
ps -eo pid,ppid,cmd,%cpu,%mem --sort=-%cpu  # Форматированный вывод
```

**Навигация в top:**
- `h` — справка
- `P` — сортировка по CPU
- `M` — сортировка по памяти
- `k` — убить процесс
- `r` — изменить приоритет (renice)
- `q` — выход

### 3. Приоритеты процессов

| Значение | Описание |
|----------|----------|
| **-20** | Максимальный приоритет (работает быстрее) |
| **0** | Приоритет по умолчанию |
| **19** | Минимальный приоритет (работает медленнее) |

**Управление приоритетами:**
```bash
# Запуск с измененным приоритетом
nice -n 10 sleep 300

# Изменение приоритета работающего процесса
renice -n 5 -p 12345      # Установить приоритет 5 для PID 12345
renice -n -10 -u sa       # Изменить приоритет всех процессов пользователя
```

### 4. Сигналы процессов

| Сигнал | Номер | Описание | Действие |
|--------|-------|----------|----------|
| **SIGTERM** | 15 | Завершить процесс (мягко) | Позволяет выполнить очистку |
| **SIGKILL** | 9 | Принудительно завершить | Немедленное завершение (жестко) |
| SIGHUP | 1 | Перезагрузка конфигурации | Перезапуск/перечитывание |
| SIGINT | 2 | Прерывание (Ctrl+C) | Завершение |
| SIGSTOP | 19 | Остановить процесс | Приостановка |
| SIGCONT | 18 | Продолжить выполнение | Возобновление |
| SIGUSR1 | 10 | Пользовательский сигнал 1 | Индивидуально |
| SIGUSR2 | 12 | Пользовательский сигнал 2 | Индивидуально |

**Отправка сигналов:**
```bash
kill -15 1234          # SIGTERM
kill -9 1234           # SIGKILL
kill -SIGTERM 1234     # Именованный сигнал
killall -9 firefox     # Завершить все процессы с именем
pkill -15 bash         # Завершить по имени (с сигналом)
```

### 5. Управление заданиями (Job Control)

| Команда | Описание | Пример |
|---------|----------|--------|
| `&` | Запуск в фоновом режиме | `sleep 300 &` |
| `jobs` | Показать фоновые задания | `jobs -l` |
| `fg` | Перевести в передний план | `fg %1` |
| `bg` | Перевести в фоновый режим | `bg %2` |
| `Ctrl+Z` | Приостановить текущее задание | Останавливает процесс |
| `nohup` | Запуск без остановки при выходе | `nohup sleep 300 &` |
| `disown` | Отсоединить задание от терминала | `disown %1` |

## Решение (эталон выполнения)

```bash
# ============ ЗАДАНИЕ 1 ============
ps aux
ps -eo pid,ppid,pcpu,pmem,cmd
ps -eo pid,pcpu,pmem,cmd --sort=-pcpu | head -6
pgrep bash
ps -u sa
pstree -p 1

# Интерактивный top
top
# Нажмите M (сортировка по памяти)
# Нажмите P (сортировка по CPU)
# Нажмите k, введите PID (например, процесса sleep), выберите сигнал 15
# Нажмите q (выход)

# ============ ЗАДАНИЕ 2 ============
# Запускаем нагрузку
dd if=/dev/zero of=/dev/null &
PID=$!
echo "PID: $PID"

# Проверяем приоритет
ps -o pid,ni,cmd -p $PID

# Изменяем приоритет через top
top
# Нажмите r, введите PID, введите новое значение nice (10)

# Проверяем
ps -o pid,ni,cmd -p $PID

# Изменяем через renice
renice -n 15 -p $PID
ps -o pid,ni,cmd -p $PID

# Запуск с указанным приоритетом
nice -n 10 dd if=/dev/zero of=/dev/null &

# Сравниваем приоритеты
ps -eo pid,ni,cmd | grep dd

# ============ ЗАДАНИЕ 3 ============
# Запускаем и отправляем SIGTERM
sleep 300 &
PID=$!
kill -15 $PID
ps -p $PID
# Должен быть убит

# Запускаем и отправляем SIGKILL
sleep 300 &
PID=$!
kill -9 $PID
ps -p $PID
# Сразу убит

# Интерактивный запуск и Ctrl+Z
sleep 300
# Нажмите Ctrl+Z
ps -o pid,stat,cmd | grep sleep
jobs
bg %1
fg %1
# Ctrl+C для завершения

# killall
killall sleep

# ============ ЗАДАНИЕ 4 ============
sleep 300 &
sleep 600 &
sleep 900 &
jobs
jobs -l
fg %2
# Ctrl+Z
bg %2
kill %3
jobs

# Использование nohup
nohup sleep 1000 &
ps aux | grep "sleep 1000"
jobs

# disown
sleep 400 &
jobs
disown %1
jobs
ps aux | grep "sleep 400"

# ============ ЗАДАНИЕ 5 ============
# Создаем лог для tail
while true; do date >> /tmp/log.txt; sleep 1; done &
PID_LOG=$!

# Запускаем tail
tail -f /tmp/log.txt
# Ctrl+Z
ps -o pid,stat,cmd | grep tail
# Возобновляем
bg %1

# Отправляем сигналы
pgrep tail
pkill -15 tail
pkill sleep

# ============ ЗАДАНИЕ 6 ============
# Процессы SSH
ps aux | grep sshd

# PID 1 принадлежит root
ps -p 1 -o user,pid,cmd

# Процесс с наибольшим временем работы
ps -eo pid,etime,cmd --sort=-etime | head -5

# Количество процессов root
ps -U root | wc -l

# Процесс с максимальным использованием памяти
ps -eo pid,%mem,cmd --sort=-%mem | head -3

# Процесс с максимальным использованием CPU
ps -eo pid,%cpu,cmd --sort=-%cpu | head -3

# PID systemd
pidof systemd

# Файлы, открытые процессом 1 (требует lsof)
lsof -p 1 2>/dev/null | head -10

# ============ ЗАДАНИЕ 7 ============
dd if=/dev/zero of=/dev/null &
yes > /dev/null &
sleep 500 &
jobs -l

# Определяем PID
JOBS=$(jobs -l)
echo "$JOBS"

# Изменяем приоритеты
renice -n 15 -p $(pgrep -n dd)
renice -n 5 -p $(pgrep -n yes)

# Мониторим в top
top

# Убиваем процессы
pkill -15 yes
pkill -9 dd
jobs

# Запускаем поиск
find / -name "*.txt" > /dev/null 2>&1 &
kill %1
jobs

# ============ ЗАДАНИЕ 8 ============
dd if=/dev/zero of=/dev/null &
dd if=/dev/zero of=/dev/null &
ps -eo pid,ni,cmd | grep dd

# Изменяем приоритеты
renice -n 10 -p $(pgrep -n dd | head -1)
renice -n 15 -p $(pgrep -n dd | tail -1)

# Запускаем top для наблюдения
top

# Изменяем приоритет для всех процессов пользователя
renice -n 5 -u sa
ps -eo pid,ni,user,cmd | grep sa | head -10
```
