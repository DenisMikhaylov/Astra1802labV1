## Теоретическая справка

### 1. Перенаправление потоков ввода/вывода

В Linux каждый процесс имеет три стандартных потока:

| Поток | Дескриптор | Описание | Перенаправление |
|-------|-----------|----------|-----------------|
| stdin | 0 | Стандартный ввод (клавиатура) | `<` |
| stdout | 1 | Стандартный вывод (экран) | `>` (перезапись), `>>` (добавление) |
| stderr | 2 | Стандартный вывод ошибок | `2>`, `2>>` |

**Примеры:**
```bash
# Перенаправление вывода в файл
echo "Hello" > file.txt

# Добавление в конец файла
echo "World" >> file.txt

# Перенаправление ошибок
ls /nonexist 2> error.log

# Перенаправление и stdout, и stderr
command > output.log 2>&1
command &> output.log

# Использование stdin из файла
sort < file.txt

# Конвейеры (pipe) - передача stdout одной команды как stdin другой
cat file.txt | grep "pattern"
```

### 2. Команды для просмотра текстовых файлов

| Команда | Описание | Пример |
|---------|----------|--------|
| `cat` | Вывод всего файла | `cat file.txt` |
| `less` | Постраничный просмотр с навигацией | `less file.txt` |
| `more` | Постраничный просмотр (проще) | `more file.txt` |
| `head` | Первые N строк (по умолчанию 10) | `head -20 file.txt` |
| `tail` | Последние N строк | `tail -30 file.txt` |
| `tail -f` | Отслеживание изменений в реальном времени | `tail -f log.txt` |

**Навигация в less:**
- `Пробел` / `PageDown` — следующая страница
- `b` / `PageUp` — предыдущая страница
- `/` — поиск вперед
- `?` — поиск назад
- `n` — следующий результат поиска
- `N` — предыдущий результат поиска
- `q` — выход

### 3. Команды для парсинга текста

#### grep — поиск по шаблону

| Опция | Описание | Пример |
|-------|----------|--------|
| `-i` | Игнорировать регистр | `grep -i "error" log.txt` |
| `-r` | Рекурсивный поиск | `grep -r "function" ./src/` |
| `-v` | Инвертировать (исключить) | `grep -v "debug" log.txt` |
| `-c` | Количество совпадений | `grep -c "error" log.txt` |
| `-n` | Номера строк | `grep -n "error" log.txt` |
| `-l` | Только имена файлов | `grep -l "TODO" *.txt` |
| `-E` | Расширенные регулярные выражения | `grep -E "error|warning" log.txt` |
| `-A` | После совпадения (After) | `grep -A 3 "error" log.txt` |
| `-B` | До совпадения (Before) | `grep -B 2 "error" log.txt` |
| `-C` | Вокруг совпадения (Context) | `grep -C 2 "error" log.txt` |

#### sed — потоковый редактор

| Действие | Синтаксис | Пример |
|----------|-----------|--------|
| Замена | `s/old/new/g` | `sed 's/foo/bar/g' file.txt` |
| Удаление строк | `d` | `sed '/pattern/d' file.txt` |
| Вывод строк | `p` | `sed -n '/pattern/p' file.txt` |
| Вставка | `i` (до), `a` (после) | `sed '/pattern/a new line' file.txt` |
| Редактирование в файле | `-i` | `sed -i 's/old/new/g' file.txt` |

**Примеры:**
```bash
# Замена первого совпадения в строке
sed 's/foo/bar/' file.txt

# Глобальная замена
sed 's/foo/bar/g' file.txt

# Удаление пустых строк
sed '/^$/d' file.txt

# Удаление строк с комментариями
sed '/^#/d' file.txt

# Сохранение в новый файл
sed 's/foo/bar/g' file.txt > newfile.txt
```

#### awk — язык обработки текста

| Концепция | Описание | Пример |
|-----------|----------|--------|
| `$0` | Вся строка | `awk '{print $0}' file.txt` |
| `$1, $2, ...` | Поля строки | `awk '{print $1, $3}' file.txt` |
| `FS` | Разделитель полей | `awk -F: '{print $1}' /etc/passwd` |
| `NR` | Номер строки | `awk '{print NR, $0}' file.txt` |
| `NF` | Количество полей | `awk '{print NF}' file.txt` |
| Условия | Фильтрация | `awk '$3 > 100' data.txt` |
| Суммирование | Агрегация | `awk '{sum+=$1} END{print sum}'` |

**Примеры:**
```bash
# Вывести первый столбец
awk '{print $1}' file.txt

# Использование разделителя : (как в /etc/passwd)
awk -F: '{print $1, $3}' /etc/passwd

# Вывести строки, где второй столбец > 100
awk '$2 > 100' data.txt

# Суммирование чисел в первом столбце
awk '{sum+=$1} END{print sum}' numbers.txt

# Форматированный вывод
awk '{printf "%-10s %5d\n", $1, $2}' data.txt
```

### 4. Текстовые редакторы

#### vim (vi improved)

| Режим | Действие |
|-------|----------|
| **Командный режим** (ESC) | Навигация, копирование, удаление |
| **Режим вставки** (i, a, o) | Ввод текста |
| **Режим командной строки** (:) | Сохранение, поиск, замена |

**Основные команды vim:**
- `i` — переход в режим вставки (перед курсором)
- `a` — переход в режим вставки (после курсора)
- `o` — новая строка снизу
- `O` — новая строка сверху
- `:w` — сохранить
- `:q` — выйти
- `:wq` — сохранить и выйти
- `:q!` — выйти без сохранения
- `dd` — удалить строку
- `yy` — копировать строку
- `p` — вставить
- `:/pattern` — поиск
- `:%s/old/new/g` — замена во всем файле

#### nano

| Команда | Описание |
|---------|----------|
| `Ctrl+O` | Сохранить |
| `Ctrl+X` | Выйти |
| `Ctrl+W` | Поиск |
| `Ctrl+\\` | Замена |
| `Ctrl+K` | Вырезать строку |
| `Ctrl+U` | Вставить |
| `Ctrl+G` | Справка |

#### mcedit (редактор Midnight Commander)

| Команда | Описание |
|---------|----------|
| `F2` | Сохранить |
| `F10` | Выйти |
| `F3` | Выделить блок |
| `F5` | Копировать |
| `F6` | Переместить |
| `F7` | Поиск |
| `F8` | Удалить |

# ============ ЗАДАНИЕ 1 ============

Hello world
Hello Linux
Hello Unix
Goodbye Windows
Goodbye MacOS
